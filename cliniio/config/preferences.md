# Preferences

## Founder Preferences

### Communication
- Lead with the decision or finding, not the preamble
- Use bullet points over paragraphs for status updates
- Flag founder decisions clearly with `⚠️ FOUNDER DECISION NEEDED:`
- Keep daily briefings to 3–5 items max

### Work Style
- Prefer smallest viable scope for features — "what's the minimum that solves the problem?"
- Classify all work into roadmap tiers before starting (hotfix / weekly patch / quarterly feature)
- Surface blockers immediately — don't work around them silently
- Ask rather than assume when priority is ambiguous

### Output Format
- Compliance checklists: plain language, source citations, organized for clinic managers
- Specs: structured (problem → solution → stories → criteria → scope → dependencies)
- Release notes: jargon-free, written for clinic managers
- Strategic briefs: options with trade-offs, always include "do nothing" option

### Review Requirements
- All public-facing content: founder review required
- All outreach: founder approval on prospect list before drafting
- All specs: founder sign-off before development
- All pricing/positioning changes: founder decision

### Provincial Context
- Primary provinces: Ontario, BC
- Expanding to: Alberta
- Always note provincial variation in compliance guidance
- Do not generate guidance for unmapped provinces
